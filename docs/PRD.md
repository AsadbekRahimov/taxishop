# TaxiShop — Product Requirements Document (PRD)

## 1. Обзор продукта

**TaxiShop** — e-commerce платформа, позволяющая таксистам продавать товары пассажирам прямо из автомобиля. Водитель авторизуется в системе, видит свой ассортимент (товары, загруженные в машину) и оформляет заказы от имени клиентов. Администратор управляет каталогом, водителями и заказами через панель Filament.

**Целевая аудитория:**
- Водители такси (основные пользователи фронтенда)
- Администраторы (управление через админ-панель)
- Конечные покупатели (пассажиры — не имеют прямого доступа к системе)

## 2. Текущий функционал

### 2.1 Аутентификация и роли

| Функция | Описание |
|---------|----------|
| Вход по логину | Водители и администраторы входят через поле `login` (не email) |
| Роли | `driver` — водитель, `admin` — администратор |
| Флаг активности | Неактивные пользователи (`is_active = false`) не могут войти |
| Remember me | Поддержка «запомнить меня» |
| Throttle | Ограничение 5 попыток входа в минуту |

### 2.2 Каталог товаров

| Функция | Описание |
|---------|----------|
| Категории | Иерархические (parent → children), с иконками и сортировкой |
| Товары | Название, описание, цена, старая цена (зачёркнутая), главное изображение, доп. изображения |
| Slug-маршруты | Каждый товар и категория имеют уникальный slug для ЧПУ |
| Наличие у водителя | Отображается количество товара в машине текущего водителя |
| Флаг доставки | Товар может быть помечен как «доставляемый» (`is_deliverable`) |
| Пагинация | 12 товаров на страницу в категории и поиске |
| Сортировка | По популярности (кол-во заказов), цене ↑/↓, новизне |

### 2.3 Поиск

| Функция | Описание |
|---------|----------|
| Текстовый поиск | По названию товара (LIKE, регистронезависимый) |
| Пагинация | 12 результатов на страницу |
| Наличие | Показывается количество в стоке водителя |

### 2.4 Корзина

| Функция | Описание |
|---------|----------|
| Хранение | В сессии (не в БД) |
| Добавление | Указание товара, количества и способа оплаты |
| Обновление | Изменение количества; при qty ≤ 0 товар удаляется |
| Удаление | Удаление отдельного товара |
| Подсчёт | Итоговая сумма и количество позиций |
| Бейдж | Количество товаров отображается в шапке сайта |

### 2.5 Оформление заказа (Checkout)

| Функция | Описание |
|---------|----------|
| Данные клиента | Имя, телефон (формат +998, Узбекистан) |
| Способы оплаты | `cash` — наличные, `qr` — QR-код, `delivery` — доставка |
| Адрес доставки | Обязателен при способе оплаты `delivery` |
| Номер заказа | Формат `TS-XXXXXX` (авто-генерация) |
| Статусы заказа | `new` → `paid` → `delivered` / `cancelled` |
| Списание стока | При оплате наличными сток водителя уменьшается сразу |
| Страница благодарности | После оформления отображается номер заказа и детали |

### 2.6 Управление стоком водителя

| Функция | Описание |
|---------|----------|
| Таблица `driver_stock` | Связь водитель ↔ товар с количеством |
| Уникальность | Один товар — одна запись на водителя |
| Декремент | При продаже за наличные количество уменьшается |
| Удаление записи | Если количество достигает 0 — запись удаляется |

### 2.7 Админ-панель (Filament v5)

| Ресурс | Возможности |
|--------|-------------|
| **Товары** | CRUD, изображения, rich-editor описание, фильтры, массовая активация/деактивация |
| **Категории** | CRUD с иерархией, иконки, порядок сортировки |
| **Водители** | CRUD аккаунтов, управление стоком через RelationManager |
| **Заказы** | Просмотр (read-only), смена статуса, фильтры по статусу/оплате/водителю/дате |
| **Дашборд** | Виджеты: статистика, график продаж, топ-товары, последние заказы |

### 2.8 Мультиязычность

| Язык | Код | Статус |
|------|-----|--------|
| Русский | `ru` | Основной |
| Английский | `en` | Поддерживается |
| Узбекский | `uz` | Поддерживается |

Переключение языка доступно из шапки сайта, язык сохраняется в сессии.

## 4. Бизнес-логика: основные сценарии

### Сценарий 1: Продажа за наличные
1. Водитель входит в систему
2. Просматривает каталог (видит только свои товары в наличии)
3. Добавляет товары в корзину
4. Оформляет заказ: вводит имя и телефон клиента, выбирает «Наличные»
5. Заказ создаётся со статусом `new`, сток водителя уменьшается
6. Отображается страница подтверждения

### Сценарий 2: Оплата QR / Доставка
1. Шаги 1-4 аналогичны, но выбирается QR или Доставка
2. Сток НЕ списывается сразу
3. Администратор меняет статус заказа вручную после получения оплаты/доставки

### Сценарий 3: Управление каталогом (админ)
1. Админ входит в `/admin`
2. Создаёт/редактирует категории и товары
3. Назначает товары водителям через DriverResource → StockRelationManager
4. Отслеживает заказы и меняет их статусы

## 5. Ограничения и валидация

- Телефон: формат `+998XXXXXXXXX` (Узбекистан)
- Slug: уникальный для товаров и категорий
- Номер заказа: уникальный, формат `TS-######`
- Удаление товара: запрещено, если есть связанные заказы (restrict)
- Удаление пользователя: запрещено, если есть связанные заказы (restrict)
- Удаление категории: каскадно удаляются товары (или restrict — зависит от настройки)
- Каскадное удаление: driver_stock при удалении водителя/товара, order_items при удалении заказа
